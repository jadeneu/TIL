## 내가 짠 코드
```python
def solution(d, budget):
    d.sort()
    cnt = 0

    for i in d:
      budget -= i
      if budget >= 0:
        cnt += 1
      else:
        break
    
    return cnt
```
처음엔 조합으로 풀었는데 런타임 에러가 나서 더 간단하게 위 코드를 짜봤다.<br>
이 코드가 더 시간 복잡도가 작다는 얘기인데, 조합이 시간 복잡도를 많이 차지한다는 것을 알게 되었다.<br>
되도록이면 조합으로 문제를 풀지 않도록 해야겠다.
